## рҹҢҹ Why Community Hero?

Most cities still handle civic complaints via phone calls, paper forms, or social media chaos. **Community Hero** replaces all of that with a single, intelligent platform:

| Without Community Hero | With Community Hero |
|---|---|
| рҹ“һ Citizens call helplines, wait on hold | рҹ“Қ Citizens drop a pin and submit in 30 seconds |
| рҹ“қ Manual data entry by staff | рҹӨ– AI categorizes, scores severity & detects duplicates |
| рҹ—ӮпёҸ Issues lost in spreadsheets | вҡЎ Real-time dashboard with live status updates |
| рҹ•җ Weeks to acknowledge a pothole | рҹҸҶ Gamified accountability speeds resolution |
| вқҢ No feedback loop for citizens | вң… Citizens track issue lifecycle end-to-end |

## вңЁ Features

<details>
<summary><b>рҹ“Қ Map-Based Reporting</b></summary>
<br/>
Citizens drop a precise pin on an interactive Leaflet map when submitting an issue. No more vague addresses вҖ” every report carries exact geolocation metadata, enabling heatmap analysis and proximity-based duplicate detection.
</details>

<details>
<summary><b>рҹӨ– AI Issue Assistant (Powered by Gemini 2.5 Flash)</b></summary>
<br/>

- **вңҚпёҸ Draft Improvement** вҖ” Automatically rewrites user-submitted titles and descriptions into professional, structured language suitable for municipal review.
- **рҹ§  Smart Analysis** вҖ” Categorizes issues (Pothole / Water Leak / Garbage / etc.), assigns severity (`Low` вҶ’ `Critical`), and routes to the responsible department.
- **рҹ”Қ Duplicate Detection** вҖ” Cross-references visual data and coordinates to surface and link duplicate reports, keeping the feed clean.
- **рҹҡЁ Fake Report Scoring** вҖ” Scores each submission for spam indicators, protecting the system's integrity.
- **рҹ’¬ Context-Aware Copilot** вҖ” A conversational assistant with full platform context, capable of summarizing issues and answering status queries.
</details>

<details>
<summary><b>рҹ“ё Media Uploads</b></summary>
<br/>
Citizens can attach photos and videos as evidence. Media is stored securely in Supabase Storage with per-bucket access policies.
</details>

<details>
<summary><b>рҹ”’ Role-Based Access Control</b></summary>
<br/>

| Role | Permissions |
|---|---|
| рҹ‘Ө **Citizen** | Submit issues, comment, vote, track own reports |
| рҹ”Қ **Verifier** | Verify or reject pending issues, add notes |
| рҹӣЎпёҸ **Admin** | Full access: resolve, generate reports, manage users |
</details>

<details>
<summary><b>рҹҸҶ Gamification & Leaderboards</b></summary>
<br/>
An XP-based points system rewards meaningful participation. Citizens earn points for submitting verified issues, verifiers earn for timely reviews, and badges like **"Civic Star"** and **"Pothole Buster"** are awarded automatically based on thresholds. Leaderboards update in real-time via Supabase Realtime.
</details>

<details>
<summary><b>вҡЎ Real-Time Dashboards</b></summary>
<br/>
Supabase Realtime WebSocket channels broadcast database mutations directly to the frontend, instantly invalidating React Query caches. The moment an issue is verified or resolved, every connected client updates вҖ” no polling, no refresh needed.
</details>

- Node.js вүҘ 18
- A [Supabase](https://supabase.com) account
- A [Google AI Studio](https://aistudio.google.com) API key (Gemini)
- [Supabase CLI](https://supabase.com/docs/guides/cli) (for Edge Functions)

---

### Step 1 вҖ” Clone & Install

```bash
git clone https://github.com/your-username/community-hero.git
cd community-hero
npm install
```

---

### Step 2 вҖ” Configure Environment

Create a `.env` file in the project root:

```env
# в”Җв”Җ Frontend (Vite) в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key

# в”Җв”Җ Backend (set in Supabase Dashboard вҶ’ Edge Functions) в”Җв”Җ
# GEMINI_API_KEY=your_google_gemini_api_key
```

> вҡ пёҸ **Never commit your `.env` file.** The `GEMINI_API_KEY` is stored as a Supabase secret and never exposed to the frontend.

---

### Step 3 вҖ” Set Up Supabase

1. **Create a new Supabase project** at [supabase.com](https://supabase.com/dashboard)

2. **Run the SQL migrations** in your Supabase SQL Editor (Project вҶ’ SQL Editor):

```sql
-- Run these in order:
-- 1. supabase_setup.sql  вҶ’ Creates profiles, issues, RLS, triggers, storage policies
-- 2. ai_migration.sql    вҶ’ Creates issue_ai_analysis table
```

3. **Create a public storage bucket** named exactly `issue-media`:
   - Go to **Storage вҶ’ Create Bucket**
   - Name: `issue-media`
   - Public: вң… enabled

---

### Step 4 вҖ” Deploy Edge Functions

```bash
# Login to Supabase CLI
supabase login

# Link to your project
supabase link --project-ref your-project-ref

# Set secrets (Gemini API key)
supabase secrets set GEMINI_API_KEY=your_google_gemini_api_key

# Deploy the AI agent function
supabase functions deploy ai-civic-agent
```

---

### Step 5 вҖ” Start Development Server

```bash
npm run dev
```

Open [http://localhost:5173](http://localhost:5173) in your browser.


Community Hero is built with a **security-first** architecture:

| Mechanism | Description |
|---|---|
| рҹ”җ **Row Level Security** | All PostgreSQL tables enforce RLS. Only authenticated users can insert. Mutations are gated by role (`created_by`, `verifier`, `admin`). |
| рҹӣЎпёҸ **Trigger Protection** | Profile points cannot be modified by frontend clients directly. A DB trigger (`protect_profile_points`) intercepts all point mutations and routes them through a secure RPC (`increment_points`). |
| рҹ”‘ **JWT Verification** | The `ai-civic-agent` Edge Function validates the Supabase JWT on every request, rejecting unauthorized calls before any AI processing occurs. |
| рҹӘЈ **Storage Policies** | The `issue-media` bucket has per-object access policies вҖ” only the authenticated uploader or admins can delete their media. |

## рҹҢҚ Deployment

### Frontend вҖ” Vercel (Recommended)

```bash

# Build Command:  npm run build
```

```toml

  publish = "dist"

  to = "/index.html"
  status = 200
```

